# java-lotto-precourse
# 로또 프로그램

## 프로젝트 개요

간단한 로또 발매기를 구현하는 프로젝트입니다. 사용자가 구입 금액을 입력하면 해당 금액만큼 로또를 발행하고, 당첨 번호와 비교하여 당첨 내역 및 수익률을 계산합니다.

## 기능 요구 사항

### 입력 또는 객체
- 로또 구입 금액을 입력 받는다.
- 구입 금액에 해당하는 만큼 로또를 발행해야 한다.
- 로또의 1장 가격은 1,000원이다.
- 당첨 번호와 보너스 번호를 입력받는다.

### 로또 기능
- 로또 번호의 숫자 범위는 1~45까지이고 1개의 로또를 발행할 때 중복되지 않는 6개의 랜덤 숫자를 출력한다.
- 6개와 중복되지 않는 보너스 번호 1개를 출력한다.
- 입력으로 받는 입력값과 일치하는 랜덤 일치수를 구한다.
- 이 때 보너스 번호와 6개의 번호는 구분한다.
- 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 로또 게임을 종료한다.

### 예외 처리
- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException 을 발생시킨다.
  - 1000원 단위인가?
  - 쉼표로 기준으로 입력 번호를 입력하였는가?
- "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.
  - Exception이 아닌 IllegalArgumentException, IllegalStateException 등과 같은 명확한 유형을 처리한다.

| 등수 | 당첨 조건 | 당첨 금액 |
|------|-----------|-----------|
| 1등 | 6개 번호 일치 | 2,000,000,000원 |
| 2등 | 5개 번호 + 보너스 번호 일치 | 30,000,000원 |
| 3등 | 5개 번호 일치 | 1,500,000원 |
| 4등 | 4개 번호 일치 | 50,000원 |
| 5등 | 3개 번호 일치 | 5,000원 |

**로또 구입 금액**
- 구입 금액은 1,000원 단위로 입력받습니다.
- 1,000원으로 나누어 떨어지지 않는 경우 예외 처리합니다.

```
14000
```

**당첨 번호**
- 번호는 쉼표(,)를 기준으로 구분합니다.

```
1,2,3,4,5,6
```

**보너스 번호**

```
7
```

### 출력

**발행한 로또 수량 및 번호**
- 로또 번호는 오름차순으로 정렬하여 보여줍니다.

```
8개를 구매했습니다.
[8, 21, 23, 41, 42, 43] 
[3, 5, 11, 16, 32, 38] 
[7, 11, 16, 35, 36, 44] 
[1, 8, 11, 31, 41, 42] 
[13, 14, 16, 38, 42, 45] 
[7, 11, 30, 40, 42, 43] 
[2, 13, 22, 32, 38, 45] 
[1, 3, 5, 14, 22, 45]
```

**당첨 내역**

```
당첨 통계
---
3개 일치 (5,000원) - 1개
4개 일치 (50,000원) - 0개
5개 일치 (1,500,000원) - 0개
5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
6개 일치 (2,000,000,000원) - 0개
총 수익률은 62.5%입니다.
```

**수익률 계산**
- 수익률은 소수점 둘째 자리에서 반올림합니다.
- 예시: 100.0%, 51.5%, 1,000,000.0%

**예외 상황 에러 메시지**

```
[ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.
```

## 프로그래밍 요구 사항

### 기본 요구 사항
- JDK 21 버전에서 실행 가능해야 합니다.
- 프로그램 실행의 시작점은 `Application`의 `main()`입니다.
- `build.gradle` 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않습니다.
- 프로그램 종료 시 `System.exit()`를 호출하지 않습니다.
- 자바 코드 컨벤션을 지키면서 프로그래밍합니다. (기본적으로 Java Style Guide를 원칙으로 합니다)

### 코드 작성 규칙
- **indent depth**: 3이 넘지 않도록 구현합니다. 2까지만 허용합니다.
    - 힌트: indent depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하는 것입니다.
- **3항 연산자**: 사용하지 않습니다.
- **함수(메서드) 길이**: 15라인을 넘어가지 않도록 구현합니다.
- **함수(메서드) 책임**: 한 가지 일만 하도록 최대한 작게 만듭니다.
- **else 예약어**: 사용하지 않습니다.
    - `switch/case`도 허용하지 않습니다.
    - 힌트: if 조건절에서 값을 return하는 방식으로 구현하면 else를 사용하지 않아도 됩니다.
- **Java Enum**: 적용하여 프로그램을 구현합니다.

### 테스트 요구 사항
- JUnit 5와 AssertJ를 이용하여 정리한 기능 목록이 정상적으로 작동하는지 테스트 코드로 확인합니다.
- 구현한 기능에 대한 단위 테스트를 작성합니다. 단, UI(System.out, System.in, Scanner) 로직은 제외합니다.
- 단위 테스트 작성이 익숙하지 않다면 `LottoTest`를 참고하여 학습한 후 테스트를 작성합니다.

## Lotto 클래스

제공된 `Lotto` 클래스를 사용하여 구현해야 합니다.

### 제약 사항
- `Lotto`에 `numbers` 이외의 필드(인스턴스 변수)를 추가할 수 없습니다.
- `numbers`의 접근 제어자인 `private`은 변경할 수 없습니다.
- `Lotto`의 패키지를 변경할 수 있습니다.

### 코드

```java
public class Lotto {
    private final List<Integer> numbers;

    public Lotto(List<Integer> numbers) {
        validate(numbers);
        this.numbers = numbers;
    }

    private void validate(List<Integer> numbers) {
        if (numbers.size() != 6) {
            throw new IllegalArgumentException("[ERROR] 로또 번호는 6개여야 합니다.");
        }
    }

}
```
